# Правила организации репозиториев

## Небольшие проекты (менее трех файлов)
1. Должен быть только один репозиторий для проектов с менее чем тремя файлами в проекте.
2. Формат названия репозитория: simple-<язык>-scripts, где <язык> - основной язык программирования в проекте.
3. Если репозитория еще нет, создаем его из готового шаблона, имя шаблона simple-scripts.
4. Данный репозиторий должен быть непубличным. Если хочешь поделиться кодом с публикой,
создавай нормальный проект, где будет более трех файлов (код, описание, тесты, документация, примеры и т.д.)

## Обычные проекты
1. Вспоминаем, по работе ли создан проект, или нет. Если по работе, то непубличный репозиторий.
2. Критически оцениваем ценность кода для публики. Если в сети полно такого же мусора, то непубличный репозиторий.
3. Иначе можно публичный репозиторий, но обязательно с подробным описанием проекта и примерами использования.
