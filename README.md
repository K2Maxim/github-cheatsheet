# Правила организации репозиториев

## Окончания строк
Соблюдаем порядок в окончаниях строк.
Если проект можно использовать только на Windows (C#, VBA, VB.NET, F#), используем окончание строк Windows (CrLf - "\r\n").
Во всех остальных случаях используем окончание строк Linux (Lf - "\n").

Для помощи в соблюдении порядка создан проект LineNormalizer.
Чтобы не мучаться с ручным запуском нормализатора окончаний из консоли, добавим запись в реестр.
- Ветка: HKEY_CLASSES_ROOT\Directory\shell (если нет админа, то HKEY_CURRENT_USER\Software\Classes\Directory\shell)
- Добавим подраздел, например: HKEY_CURRENT_USER\SOFTWARE\Classes\Directory\shell\line_norm\command
- Пропишем свойство HKEY_CURRENT_USER\SOFTWARE\Classes\Directory\shell\line_norm:

      Нормализация строк

- Пропишем свойство HKEY_CURRENT_USER\SOFTWARE\Classes\Directory\shell\line_norm\command:

      "C:\Users\maxim.tolstikov\source\repos\LineNormalizer\LineNormalizer\bin\Debug\net48\LineNormalizer.exe" "%1"

- После этого при щелчке правой кнопки на папку в контекстом меню появится пункт "Нормализация строк", который делает окончания строк всех файлов в папке одинаковыми (Lf).

## Небольшие проекты (менее трех файлов)
- Должен быть только один репозиторий для проектов с менее чем тремя файлами в проекте.
- Формат названия репозитория: simple-<язык>-scripts, где <язык> - основной язык программирования в проекте.
- Данный репозиторий должен быть непубличным. Если хотим поделиться кодом с публикой,
нужно создавать нормальный проект, где будет более трех файлов (код, описание, тесты, документация, примеры и т.д.)
- Если репозитория еще нет, создаем его из готового шаблона, имя шаблона simple-scripts.
- После того, как создан репозиторий, создаем локальную папку репозитория на диске D:\
- Далее:
  1. копируем из Проводника Windows путь к созданной папке
  2. вставляем в SublimeText
  3. выбираем из контексного меню Инструменты Картотеки -> Работа c Git -> Преобразовать формат пути к файлу из WIndows в Linux
  4. преобразованный путь вставляем в команду cd Git Bash: cd <преобразованный путь>
- Далее:

      git init
      git remote add origin https://github.com/K2Maxim/simple-<язык>-scripts.git
      git pull origin main --set-upstream

- Создаем папки под каждую заготовку кода, копируем в папку заготовку и добавляем README.md с описанием, что делает эта заготовка.
- Также в корне репозитория редактируем README.md, добавив упоминания и краткие описания всех заготовок кода в репозитории.

## Обычные проекты
- Вспоминаем, по работе ли создан проект, или нет. Если по работе, то непубличный репозиторий.
- Критически оцениваем ценность кода для публики. Если в сети полно такого же мусора, то непубличный репозиторий.
- Иначе можно публичный репозиторий, но обязательно с подробным описанием проекта и примерами использования.
